Features to Be Tested

  • Doesn't crash
    • MED : Performance on potentially outdated systems, must ensure the IDE does not depend on certain system-specific properties
    • HI : Resistance to nonsensical and/or malicious combinations of user input
  • Ease of use
    • HI : Amount of navigation needed to reach any given option, on average, must be low
    • MED : Uses hotkey mappings similar to industry standard, or jEdit's use
  • Looks familiar
    • HI : Gui can be navigated in the same way that Office applications can, with elements such as the Ribbon Menu
    • MED : GUI looks like an office application (Tested manually by a few randomly selected employees)
